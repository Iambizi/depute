## Overview

Minimal execution steering — pause, resume, stop, retry. Intentionally kept simple. Extend via the actions slot for additional controls.

## Usage

```tsx
import { useState } from 'react';
import { RunControls } from '@/components/RunControls';
import type { RunState } from '@/types/ax-common';

export function App() {
  const [state, setState] = useState<RunState>('running');

  return (
    <div className="p-4 border border-zinc-200 dark:border-zinc-800 rounded-lg max-w-sm">
      <RunControls 
        state={state} 
        onStart={() => setState('running')}
        onPause={() => setState('paused')}
        onStop={() => setState('stopped')}
        onRetry={() => setState('running')}
        showLabel={true}
      />
    </div>
  );
}
```

## Props

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `state` | `RunState` | **Required** | Current execution state (`'idle' \| 'running' \| 'paused' \| 'completed' \| 'failed' \| 'stopped'`). |
| `onStart` | `() => void` | `undefined` | Callback fired when user clicks start or resume. |
| `onPause` | `() => void` | `undefined` | Callback fired when user clicks pause. |
| `onStop` | `() => void` | `undefined` | Callback fired when user clicks stop (cancel execution entirely). |
| `onRetry` | `() => void` | `undefined` | Callback fired when user clicks retry after a failure state. |
| `showLabel` | `boolean` | `false` | Whether to display a text label showing the current state next to the pulsing indicator. |
| `actions` | `React.ReactNode` | `undefined` | Slot for additional custom action buttons (e.g., an Undo button). |
| `className` | `string` | `undefined` | Additional CSS class for the root element. |

## Design Rationale

**1. The "Big Red Button" Principle**  
When an agent is operating autonomously, the human supervisor must always have an immediate, obvious way to halt it. `RunControls` provides this physical-feeling override switch.

**2. State-Dependent Visibility**  
The controls automatically adapt to the `state` prop. If the agent is already `completed`, the pause and stop buttons disappear. If it is `failed`, the retry button appears. This prevents impossible state transitions.

**3. Extensibility via Slots**  
While `RunControls` provides the core lifecycle buttons, the `actions` slot allows developers to bolt on domain-specific controls (like "Undo" or "Fork Branch") without forking the component code.
