## What is Depute?

*Depute (verb meaning to appoint or delegate).*

**Depute** is a React component library for Agentic Experience (AX) design — purpose-built UI primitives for AI agent supervision and human oversight.

When an AI agent runs in your product, users need to see what's happening — not raw logs, but structured UI for delegation, trust, and control.

## Why Depute?

Most UI libraries build for CRUD apps. Depute builds for the moment *after* you hand off work to an agent:

- The agent proposes a plan → [**Plan Card**](/docs/components/plan-card) shows it before execution
- The agent wants to send email → [**Approval Gate**](/docs/components/approval-gate) blocks until the user approves
- The agent is running → [**Run Controls**](/docs/components/run-controls) let the user pause or stop at any time
- Multiple agents are coordinating → [**Orchestrator View**](/docs/components/orchestrator-view) shows who's managing whom

**A toolkit for both Builders and Auditors:**
For Product Engineers, Depute provides the UI to stop users from feeling anxious and losing context during long autonomous runs. 
For Security & Risk Teams, Depute is compliance infrastructure. Under emerging governance standards—like the **Singapore IMDA Agentic AI Governance Framework (Jan 2026)**—providing escalation paths, defining UI checkpoints for high-stakes actions, and making humans "meaningfully accountable" is a primary regulatory expectation. Depute helps you operationalize those governance requirements in code.

## Defeating Cognitive Failures

Human-automation research shows that humans are terrible at passively monitoring highly autonomous systems. When working with AI agents, users fall into predictable cognitive traps. Depute maps its primitives directly against these known failures:

- **Automation Bias:** Users blindly rubber-stamp agent actions without scrutiny → *Mitigated by an [**Automation Bias Alert**](/docs/components/automation-bias-alert) and [**Confidence Meter**](/docs/components/confidence-meter)*
- **Execution Amnesia:** Users lose track of the total "blast radius" of a massive agent session → *Mitigated by a [**Session Overview**](/docs/components/session-overview) and [**State Diff**](/docs/components/state-diff)*
- **Responsibility Diffusion:** Users approve actions assuming the machine is accountable → *Mitigated by a [**Decision Record**](/docs/components/decision-record) and [**Binding Approval**](/docs/components/binding-approval)*

## Mitigating Systemic Threats

While Cognitive Failures describe UX traps, Systemic Threats describe catastrophic operational failure modes increasingly targeted by international AI legislation (e.g., IMDA Systemic Threats). Depute's components function as active algorithmic firebreaks:

- **Cascading Failures:** An agentic error pollutes downstream autonomous systems rapidly → *Contained by a [**Swarm Monitor**](/docs/components/swarm-monitor) and [**Orchestrator View**](/docs/components/orchestrator-view)*
- **Hallucinated Planning:** An agent proposes a destructive or nonsensical sequence of tools → *Exposed by a [**Plan Card**](/docs/components/plan-card) and [**Confidence Meter**](/docs/components/confidence-meter)*
- **Rogue Tool Use:** An agent attempts an irreversible action out-of-bounds → *Blocked by an [**Approval Gate**](/docs/components/approval-gate) and [**Tool Trace**](/docs/components/tool-trace)*
- **Unpredictable Outcomes:** A run completes with thousands of opaque, unlogged side effects → *Audited with a [**State Diff**](/docs/components/state-diff) and [**Session Overview**](/docs/components/session-overview)*

## Use-Case Index (Problem-First)

Not sure which component to use? **Press `⌘K` to search by problem.** The docs are mapped to UX challenges (e.g., search for "cognitive load" or "context switch").

Or start here:
- *"I need to safely switch agent focus across a swarm"* → Use an [**Orchestrator View**](/docs/components/orchestrator-view) + [**Shared Context Ledger**](/docs/components/shared-context-ledger)
- *"I need to stop an agent from making an unrecoverable mistake"* → Use an [**Approval Gate**](/docs/components/approval-gate) + [**Tool Trace**](/docs/components/tool-trace)
- *"I need to show the user what the agent is thinking"* → Use a [**Plan Card**](/docs/components/plan-card) + [**Confidence Meter**](/docs/components/confidence-meter)
- *"I need to prevent users from blindly rubber-stamping approvals"* → Use an [**Automation Bias Alert**](/docs/components/automation-bias-alert) wrapping an [**Approval Gate**](/docs/components/approval-gate)
- *"I need a binding cryptographic signature for a ledger mutation"* → Use a [**Binding Approval**](/docs/components/binding-approval) + [**Transaction Receipt**](/docs/components/transaction-receipt)
- *"I need to roll back a sequence of 40 API calls"* → Use a [**Rollback Timeline**](/docs/components/rollback-timeline) + [**State Diff**](/docs/components/state-diff)

## Distribution model

Components are delivered via CLI copy-paste (like shadcn/ui). The source lands in your repo — you own it, you can read it, you can modify it.

```bash
npx ax-depute@latest add approval-gate
```

No lock-in. No black-box npm package the agent can't inspect.

## The AI Skill Layer

You aren't the only one building this app—your AI agent is too. Depute ships with a native **Skill Layer** built for Claude Code, Cursor, and Copilot.

When you install a component, you get the UI code for the human *and* the structural reasoning instructions for your AI agent. The skill teaches your agent exactly when to use an `ApprovalGate` versus a `RunControls` boundary, ensuring the UI your agent helps you build is the same UI your agent knows how to use.

```bash
npx ax-depute@latest init
```

This command drops a `skills/depute/SKILL.md` file into your project, automatically giving Claude Code, Cursor, or Copilot deep intelligence on how to use Depute's primitives in your specific codebase.
